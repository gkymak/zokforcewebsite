# ZokForce Design Thinking Skill

When activated, perform a "Visitor & Purpose Analysis" before writing any code.

## 1. Visitor Identification
Identify which visitor persona is being served:
- **Prospective Client**: Exploring AI consulting services, needs trust signals
- **Technical Decision Maker**: Evaluating capabilities, wants case studies and methodology
- **Existing Client**: Looking for contact info or new service offerings
- **Talent / Partner**: Assessing company culture and expertise

## 2. Content Purpose
Define the "Job to be Done" — every section must answer a visitor question:
- "Can these people solve MY problem?" → Case Studies
- "What exactly do they offer?" → Services
- "Are they credible?" → About, Stats, Blog
- "How do I get started?" → Contact, CTA

## 3. Brand Alignment
Ensure all content reflects ZokForce's brand pillars:
- **AI Excellence**: Technical depth, not buzzwords
- **Customer Success**: Results-oriented, metrics-driven
- **Ethical AI**: Responsible, transparent
- **Digital for All**: Accessible, multilingual

## 4. Internationalization (i18n) by Design
All features MUST be designed with multi-language support:
- Consider text expansion (Chinese characters are more compact; English is wider)
- All text must support EN, zh-TW, zh-CN
- Test layout in all 3 languages before completing

## 5. Mandatory Planning Workflow
**🚨 STRICT ENFORCEMENT 🚨**
1. **Analyze**: Identify target persona and content purpose
2. **Design**: Sketch the section structure (which layout pattern from `zokforce-layout-consistency`)
3. **Plan i18n**: List all new translation keys needed
4. **Request Approval**: Present plan to user before writing code
